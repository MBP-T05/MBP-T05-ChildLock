## 📝 유스케이스 명세서 (Use Case Specification)

### UC-1. 운전자 차일드 락 전자식 ON/OFF 제어
운전자가 스위치를 조작하여 뒷좌석 차일드 락 기능을 활성화 또는 비활성화하는 기능

* **Pre-condition (사전 조건)**
  * 차량 전원 ON
  * 차일드 락 제어 스위치 정상 동작
  * 도어 ECU 통신 가능
* **Post-condition (사후 조건)**
  * 차일드 락 상태가 ON 또는 OFF로 변경됨
  * 뒷좌석 내부 핸들 동작 상태에 반영됨
  * 상태 변경 로그 저장
* **Trigger (시작 조건)**
  * 운전자가 차일드 락 스위치를 누름

#### 🟢 Main Success Scenario (MSS)
1. 운전자가 차일드 락 버튼을 누른다.
2. 시스템이 스위치 입력을 수신한다.
3. 입력 신호 유효성을 검증한다.
4. 목표 상태(ON/OFF)를 결정한다.
5. 도어 ECU에 차일드 락 제어 명령을 전송한다.
6. 뒷좌석 내부 핸들 활성/비활성 상태를 적용한다.
7. 계기판 또는 표시등 상태를 갱신한다.
8. 상태 변경 이벤트를 로그에 기록한다.

#### 🟡 Alternative Scenario (대안 시나리오)
* 스위치 입력이 불안정한 경우 (Chattering 등), 일정 시간 내 재확인 후 처리한다.

#### 🔴 Exception Scenario (예외 시나리오)
* ECU 통신 실패 시, 계기판에 경고를 표시한 후 이전 상태를 유지(Fail-safe)한다.
